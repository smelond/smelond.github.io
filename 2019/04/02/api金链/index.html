返回结果

  • accountID： 账号ID
  • sequence： 当前账号提交序列
  • balance： 原生资产余额
  • subEntryNum： 实体数
  • inflationDest： 通胀地址

  • thresholds： 操作权限阈值
  • signers： 签名者
  • trustLines： 信任类资产信息列表
  • data： 实体数据

flags内容：

  • required： 需要发行地址进行授权
  • revocable： 允许发行地址撤销信用资产、冻结。
  • immutable： 表示账户永远无法删除

thresholds内容：

  • lowThreshold： 低级安全阈值
  • medThreshold： 中级安全阈值
  • highThreshold： 高级安全阈值
  • masterWeight： 当前账户主权重

signers内容：

  • signerAccount： 签名账户
  • weight： 权重
  • type： 预留

trustLines内容：

  • type： 资产类型
  • code： 资产编码
  • issuer： 发行方
  • limit： 限额
  • balance： 余额

二十三、Account参数

属性 含义 类型 说明
accountID 与address同义 为金链的address；即：账户公钥，账户地址
balance 金链原生资产余额 XDE余额；准确讲，以金为单位的余额信息
data 账户实体属性、数据集合 由name与value组成的对象集合
subEntryNum number of subentries 当前账户拥有的entry数量，这个数量用于计算一个账户的最小可用余额。entry类型：Signer、Trustline、Data
Thresholds 操作安全阈值 每种操作类型对应的安全阈值；定义为"highThreshold", "lowThreshold", "medThreshold"，masterWeight为当前账户权重，用于多签名。
signers 签名账户列表 给当前账户附加的签名账户；用于多签名
flags 账户标记 有三个Flag，用于标识资产发行地址。（1）Authorization required (1)即需要授权访问。即：使用这个发行地址前，需要发行地址进行授权；（2）Authorization revocable (2)即可以撤销的。即：允许发行地址撤销信用资产，并冻结；（3）Authorization immutable (4)即不可修改的（设置后就无法修改了）。即：设置此标记，表示不能再设置任何authorization flags。

二十七、Ledger参数Ledger Header

属性 含义 是否必填 类型 说明
Version 版本号 uint32 账页协议版本
Previous Ledger Hash 前一个账页Hash 前一个账页Hash
Transaction set result hash 事务集Hash Hash of the results of applying the transaction set. This data is not, strictly speaking, necessary for validating the results of the transactions. However, this data makes it easier for entities to validate the result of a given transaction without having to apply the transaction set to the previous ledger.
Bucket list hash 账页中所有对象的Hash，数据结构为bucket list
Ledger sequence 账页序列号 账页序列号
Fee pool 费用池 已经支付费用的XDE数量，这个数字将加到通货膨胀池，下一次通货膨胀运行时将设置为0，注意单位是XDE。
Inflation sequence 通货膨胀序列号 已经运行的通货膨胀的次数。
ID pool ID池 最后使用的全局ID，这些ID用于生成对象。
Maximum Number of Transactions 事务最大数 验证器能够验证的最大交易数量，如果超过数量，则验证器将包括费用最高的事务。
Base Fee 基础费用 每个操作的基础费用，单位是XDE
Base reserve 基础保证金 基础保证金
Skip list 之前的账页Hash，能跳到之前的账页，不用one by one，有四个Hash值存储，存储模50 5000 50000 or 500000四个，具体的值是skipList[0] mod(50), skipList[1] mod(5000), 以此类推。
